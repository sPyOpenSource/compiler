Glossary: AOT & JIT Compilation

AOT (Ahead-of-Time) compilation
Translating source or bytecode into machine code before execution, usually at build time. Produces a standalone executable. Lesson 1
JIT (Just-in-Time) compilation
Translating bytecode to machine code during execution, typically for frequently-executed ("hot") methods. The JVM's default mode. Lesson 1
Closed-world assumption
The assumption that all reachable code is known at build time. AOT compilers use this to devirtualize calls, inline aggressively, and eliminate dead code. Breaks with reflection or dynamic class loading. Lesson 1
Warmup
The period during which a JIT-compiled application transitions from interpreted execution to fully optimized native code. AOT binaries skip warmup entirely. Lesson 1
PGO (Profile-Guided Optimization)
A two-phase AOT strategy: (1) build an instrumented binary and run it on representative data to collect profiles; (2) rebuild with those profiles guiding optimization decisions. Bridges the JIT/AOT gap. Lesson 1
Substrate VM
The lightweight runtime embedded in GraalVM native images. Provides memory management (GC), thread scheduling, and exception handling — but no bytecode interpreter or JIT compiler. Lesson 1
Points-to analysis
A static analysis that determines which objects a reference variable can point to. Used by AOT compilers under the closed-world assumption to enable devirtualization and escape analysis. Lesson 1
Devirtualization
Replacing a virtual method call (which requires a vtable dispatch) with a direct call to a known implementation. Easy under closed-world; requires guards in a JIT. Lesson 1
Deoptimization
A JIT mechanism that rolls back optimized code when its speculative assumptions (e.g., "this call is monomorphic") prove wrong. Not available in AOT — the compiled code is fixed. Lesson 1
Native image
A self-contained executable produced by GraalVM's AOT compiler. Includes application code, JDK classes, and Substrate VM — but no JVM. Lesson 1
JVM Slot
The basic unit of storage on the JVM operand stack and in local variables. Typically 32 bits wide. Lesson 2
Two-Slot Type
Types (long, double) that occupy two consecutive slots on the stack. This requires special handling in the compiler's stack simulation. Lesson 2
XMM Register
Specialized 128-bit registers used by x86 for floating-point (SSE) and SIMD operations. Lesson 2
GPR (General Purpose Register)
Standard CPU registers (like RAX, RBX) used for integer arithmetic and memory addresses. Lesson 2
